Initial setup is simple: create a new GitHub repository named username.github.io

This can be done to create user, organization or project pages.

This is similar any other server directory at this point, in which we can drop html, css, js, and images.

Files are pushed to the site through git, which can make dev tedious without a local environment.

There is also a templating system called Jeykll, which is Ruby-based.

This is a very dev-centric environment in general.

It IS for code repositories!

But technology is part of every enterprise -- as is design. Design may be a little under-represented here, in favor of utility.

Using GitHub to host org and project sites appears to be quite common among the most active GitHub projects.

But it is not 100% by any means.

In some cases, projectname.github.io hosts the main website or a landing page.

Another common scenario is organizations using GitHub Pages to host their docs.


GitHub sites do not appear to be particularly marketing heavy.

Their focus is in line with the open source ethos, with most content geared toward:

  • Describing the service / technology
  • Presenting benefits to the user and industry
  • Providing how-to information

But in technology, that is marketing.

Templating and "blogging" are possible using Jekyll a "static site generator."

Jekyll is very similar to Sphinx, the platform that the Ansible Docs team is currently using for templating.

Jekyll supports Markdown, which could be a huge deal for the engineering and docs team if we hosted docs on GitHub.

There are, however, some detractors to the idea of hosting docs with multiple authors on GitHub, warning or merge conflicts.

Ironically, their suggestion was to use Sphinx.

This may be something that the engineering team has already explored and considered.

The takeaway is that you can build anything you want on GitHub pages.

The primary differentiator is the process for building.

  • No CMS
  • No database
  • Content authors must be familiar with using and deploying through Git.
