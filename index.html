Initial setup is simple: create a new GitHub repository named username.github.io

This can be done to create user, organization or project pages.

This is similar any other server directory at this point, in which we can drop html, css, js, and images.

Files are pushed to the site through git, which can make dev tedious without a local environment.

There is also a templating system called Jeykll, which is Ruby-based.

This is a very dev-centric environment in general.

It IS for code repositories!

But technology is part of every enterprise -- as is design. Design may be a little under-represented here, in favor of utility.

Using GitHub to host org and project sites appears to be quite common among the most active GitHub projects.

But it is not 100% by any means.

In some cases, projectname.github.io hosts the main website or a landing page.

Another common scenario is orgs using GitHub pages to host their docs.
